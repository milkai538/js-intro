## 💬 Section 1: The "Why" and "How"

Comments are lines of text that the computer completely ignores, yet they are some of the most important lines of code you will ever write.

### 💬 Prompt 1.1

**Copy and paste this prompt into Gemini:** `I am a high school web design student. Explain why professional programmers add comments to their JavaScript code. Please provide one example of a *helpful/clear* comment and one example of a *vague/unhelpful* comment so I can see the difference.`

### 📝 Think & Record

* **What is the main reason developers add comments to their code?**
* To keep an idea they have, or train of thought, written down so they can remember it later.


* **What made the example of a vague comment difficult to understand?**
* Essentially just a lack of proper communication. You'll never get through much of a conversation if you just say a few key terms only, especially if you just have no clue what the original commenter was referring to. I mean, can you just try and understand this: "Purple. man. food. kids. suits. rot."

**Copy and paste this prompt into Gemini:** `Show me the syntax for the two types of comments in JavaScript: single-line and multi-line. When would it be better to use a multi-line comment instead of just several single-line comments?`

### 📝 Think & Record

* "//" without quotes


* **What symbols allow a multi-line comment to wrap on your screen?**

## 🛠️ Section 2: Commenting Out Code

Sometimes, the best use of a comment is to temporarily hide code from the browser (not to delete the code). This is a superpower when debugging or testing your code.

### 💬 Prompt 2.1

**Copy and paste this prompt into Gemini:** `Explain the concept of **commenting out** a block of code. Imagine I am testing two different ways to define and assign a value to a JavaScript variable. How can I use comments to quickly switch between the two different lines of code without deleting either one?`

### 📝 Think & Record

* **How does commenting out code actually help you improve your code?**
* Allows you to have other ideas floating in the code without it affecting the outcome, can allow you to copy and paste these alternatives if you'd prefer them or if the code you're trying isn't working, and -- again -- it helps you to keep a train of thought going.


* **What is the keyboard shortcut in VS Code to comment out a line of code? (Ask Gemini if you don't know!)**
* "Control + /" for javascript, and no real shortcut for html
